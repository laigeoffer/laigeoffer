如果你正在参加秋招，并且简历上没有什么**亮眼的项目**可以写，那么 PmHub 绝对是一个**非常优质**的选择。

目前看了很多优质实战项目，虽说微服务项目也不少，但貌似很少有项目是带着你从单体项目一步步改造成微服务项目，我们会将单体应用微服务化的踩坑经验分享出来，让你明白**架构的演进史**！

除了这些，PmHub 还有以下亮点等着你去发现：

> - **热门技术**：采用时下企业最新的热门技术框架，如 SpringCloud-Gateway、Nacos、Sentinel等，主打一个硬核，不与真实企业项目脱节；
> - **单体与微服务**：提供单体和微服务，并能体验从单体到微服务架构的改造全过程，深入理解两种架构的优缺点；
> - **面试硬核**：设计贴近校招实际需求，提供把项目写进简历全套模板，提供简历修改及 1V1 咨询服务；
> - **代码质量**：前大厂一线开发现团队 leader 亲自下场，严格遵循代码规范和最佳实践，提高代码质量和维护性；
> - **持续集成**：提供持续集成和持续部署的完整配置，带你从 0-1 用 docker 上线项目；
> - **产品设计**：提供产品设计文档，包括产品需求、产品架构、产品原型等，[在线查看产品原型](https://lanhuapp.com/link/#/invite?sid=qxZji4oa)；
> - **企业工作流**：项目提供企业工作流系统，可以快速定制出适合自己公司的企业工作流系统。

## 02、PmHub 能让你学到什么

对于 PmHub 这个项目，我们是有野心的：

1、历来 BPM 流程是最复杂但又是每个企业都必须的，但凡项目需要涉及到多人协同，审批流转，很多人就会手足无措，不知道如何开发，而很多开源的简直没法使用，付费的又价格居高不下。我们代码全部开源，希望能让开发者用好 BPM 就像呼吸空气一样简单。

2、找工作的时候，很多小伙伴面临一堆「烂大街」的项目，经常还没面试就挂了简历， 项目亮点也不足，就很吃亏，虽然 GitHub 和码云上已经有了不少优秀的开源项目，但**大多数没有成熟且体系化的教程**，总不能直接下载到本地跑婴喜爱 main 方法就算学习了吧？

3、微服务项目有很多，但**真正有能从单体到微服务进化史的项目却很少**，只有真正经历这一过程，你对微服务的理解才更透彻。我们希望的就是授人以渔。

我们要负责到底！接下来，我们会更新一系列的教程，不仅包含项目开发文档，面试系列、还会包括 Spring、MySQL、Redis、微服务&分布式、消息队列、操作系统、计算机网络、数据结构与算法等内容。

总之一句话：**学编程，就上 PmHub**😁。

### 组件库开发

组件库源于对公共功能的封装，避免相同的功能在不同项目之间重复复制。

为了统一各个微服务可能使用到的公共内容，我们在这里规划了常用且通用的功能点，供大家去使用，以提高编码效率，我们统一放在了 pmhub-base 模块下，这将作为一个公共 jar 包供各个微服务和项目使用。

当然如果你有更好的想法，也可以联系我们将其加入到公共包中。

组件库的开发宗旨是汇总资源，更高效地提供业务敏捷开发的能力，后续的迭代也将遵循这一原则。目前，这只是一个起点，是整体规划的一部分，还有许多可以提升的空间。
